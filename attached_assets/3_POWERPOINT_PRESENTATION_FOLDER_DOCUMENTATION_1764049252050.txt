1. PROJECT OVERVIEW & ARCHITECTURE
================================================================================

APPLICATION NAME: BET Slides AI - Professional Presentation Generator

VERSION: 3.0.0 (Cloud Sync Edition)

PROBLEM SOLVED:
- Time-consuming PowerPoint presentation creation
- Skills gap in content structuring and visual design
- Finding relevant, high-quality images
- Writing effective speaker notes
- Maintaining visual consistency
- Managing multiple presentation versions

SOLUTION:
An advanced AI-powered web application that automatically generates professional
presentation decks in seconds. Combines:
- Google Gemini 2.5 Flash AI for content generation
- Pixabay API for high-quality stock images
- Text-to-speech coaching
- Firebase cloud storage
- PowerPoint export capabilities
- Runs entirely client-side with no backend required

TARGET USERS:
- Business professionals
- Educators and trainers
- Students and researchers
- Marketers and sales teams
- Conference speakers

ARCHITECTURE TYPE: Frontend-Only / Client-Side Application (Jamstack)

ARCHITECTURE PATTERN: Component-Based Single Page Application (SPA)

KEY ARCHITECTURAL DECISIONS:
- No Backend Server: Entire application runs in the browser
- Direct API Integration: Client directly calls Google Gemini, Pixabay, Firebase
- Serverless Deployment: Static hosting on Netlify
- Real-time Database: Firebase Firestore for cloud persistence
- CDN-Based Libraries: PptxGenJS loaded dynamically only when needed

DATA FLOW:
User Input → Gemini API (Content) → Pixabay API (Images) →
React State Management → Preview → Firebase (Save) / Export (PPTX/PDF)

COMPONENT ARCHITECTURE:
App (Root)
├── PresentationSession (Main Logic)
│   ├── Input Step (Topic, Files, Settings)
│   ├── Generation Step (Loading State)
│   └── Results Step
│       ├── Slide Component (Individual Slides)
│       │   └── Image Generation (Pixabay)
│       └── PresentationCoach (TTS & Speaker Notes)
├── SettingsPanel (API Key Management)
├── ContactPanel (Support Information)
├── AboutPanel (App Information)
└── Sidebar (Navigation & Project Management)

Keyword Extraction Algorithm (Lines 292-311):
// Step 1: Remove generic presentation words
cleanedPrompt = prompt.replace(/\b(professional|presentation|slide|corporate|style|showing)\b/gi, '');

// Step 2: Extract meaningful words (>3 chars)
const words = cleanedPrompt.split(/\s+/)
  .filter(w => w.length > 3)
  .filter(w => !/^(this|that|with|from)$/.test(w));

// Step 3: Take 6-8 keywords for specificity
const keywords = words.slice(0, Math.min(8, words.length)).join(' ');

// Step 4: Fetch 10 images and pick randomly
const randomIndex = Math.floor(Math.random() * data.hits.length);

- image_type: "photo" (excludes illustrations)

- orientation: "horizontal" (landscape format)
- order: "popular" (most downloaded images first)

const pcmToWav = (base64PCM, sampleRate = 24000) => {
  const binaryString = atob(base64PCM);
  const len = binaryString.length;
  const buffer = new ArrayBuffer(44 + len); // WAV header + data
  const view = new DataView(buffer);

  // Write WAV header (44 bytes)
  writeString(view, 0, 'RIFF');
  view.setUint32(4, 36 + len, true);
  writeString(view, 8, 'WAVE');
  // ... (header details)

  // Write PCM data
  const pcmData = new Uint8Array(buffer, 44);
  for (let i = 0; i < len; i++) {
    pcmData[i] = binaryString.charCodeAt(i);
  }

  return new Blob([view], { type: 'audio/wav' });
};

- Format: "WAVE"
- Subchunk1ID: "fmt "
- AudioFormat: 1 (PCM)

- SampleRate: 24000 Hz
- BitsPerSample: 16
